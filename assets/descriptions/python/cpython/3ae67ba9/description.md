Title
-----
Core header refactor: consolidate struct definitions to break cross-header dependencies

Summary
-------
Moves most struct definitions from various `pycore_*` headers into two new centralized headers (`pycore_structs.h` and `pycore_runtime_structs.h`) to eliminate circular dependencies.

Why
---
- Headers were including each other creating complex dependency chains
- Circular dependencies made compilation slower and more fragile
- Difficult to understand what depended on what

Changes
-------
**New headers created:**
- `pycore_structs.h` - common structs used across multiple modules
- `pycore_runtime_structs.h` - runtime/interpreter state structs
- `pycore_interp_structs.h` - interpreter-specific struct definitions

**Structs consolidated:**
- Monitoring: `_Py_LocalMonitors`, `_Py_GlobalMonitors`, `_PyCoMonitoringData`, ...
- GC: `PyGC_Head`, `gc_generation`, `_gc_runtime_state`, ...  
- Imports: `_import_runtime_state`, `_import_state`, ...
- Code/functions: `_py_code_state`, `_py_func_state`, `_Py_CODEUNIT`, ...
- Context/HAMT: `PyHamtObject`, `PyHamtNode_Bitmap`, ...
- Atexit: `_atexit_runtime_state`, `atexit_state`, ...
- Pending calls: `_pending_calls`, `_pending_call`, ...
- Types: `types_state`, `_types_runtime_state`, type cache structs, ...
- Unicode: `_Py_unicode_state`, `_Py_unicode_runtime_state`, ...
- Float: `_Py_float_runtime_state`, ...
- Time: `_Py_time_runtime_state`, `_PyTimeFraction`, ...
- Parser: `_parser_runtime_state`, ...
- Codecs: `codecs_state`, ...
- Dtoa: `_dtoa_state`, `Bigint`, ...
- And ~30+ more...

**Impact:**
- Headers can now include struct definition headers without pulling in entire subsystems
- Forward declarations used where full definitions not needed
- Constants moved alongside related structs (e.g., `TYPE_MAX_WATCHERS`, `CODE_MAX_WATCHERS`)
- Cleaner dependency graph

**No functional changes** - pure refactoring for maintainability.
